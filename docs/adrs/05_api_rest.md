# 5. API REST para comunicación

Fecha: Julio 3 de 2025

## Status

Aceptada.

## Contexto

Se requiere una interfaz estándar y ampliamente soportada para la comunicación entre el frontend y el backend, que permita la integración con diferentes clientes y facilite la documentación y pruebas.

## Decisión

Se adopta el uso de API REST para la comunicación entre el frontend y el backend.

## Consecuencias

Facilita la interoperabilidad, la escalabilidad y la integración futura con otros sistemas o clientes. Permite el uso de herramientas estándar para pruebas y documentación.
